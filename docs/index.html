@vbarbarosh/dd

An abstraction for working with the mouse — and, since pointer events, with touch and pen.

Every drag interaction has the same skeleton: it begins, the pointer moves, it ends. What you actually care about is how far the pointer traveled from the press point. dd owns that workflow: it attaches document-level listeners, computes deltas, survives scrolling, thresholds, and cancellation — and hands you plain numbers.

npm i @vbarbarosh/dd

Or straight from a CDN (exposes a global dd function):

<script src="https://unpkg.com/@vbarbarosh/dd@1.9.0/dist/dd.js"></script>

Demo

Drag the box. It snaps to a 10px grid and reports its position.

box.addEventListener('pointerdown', function (event) {
    const x0 = p.x;
    const y0 = p.y;
    dd({
        event,
        mixins: [dd.prevent_default(), dd.grid(10), dd.no_pointer_events()],
        move: function ({dx, dy}) {
            p.x = clamp(x0 + dx, 0, canvas.clientWidth - 50);
            p.y = clamp(y0 + dy, 0, canvas.clientHeight - 50);
            render_box(box, p);
        },
    });
});

Threshold and cancellation

This one arms after the pointer travels 20px — a stray click won't start a drag. Press ESC mid-drag (or switch windows) and it snaps back to where it started.

box.addEventListener('pointerdown', function (event) {
    const x0 = p.x;
    const y0 = p.y;
    dd({
        event,
        threshold: 20,
        mixins: [dd.prevent_default(), dd.no_pointer_events()],
        begin: function () {
            box.classList.add('dragging');
        },
        move: function ({dx, dy}) {
            p.x = clamp(x0 + dx, 0, canvas.clientWidth - 50);
            p.y = clamp(y0 + dy, 0, canvas.clientHeight - 50);
            render_box(box, p);
        },
        end: function (ctx) {
            box.classList.remove('dragging');
            if (ctx.canceled) {
                p.x = x0;
                p.y = y0;
                render_box(box, p);
            }
        },
    });
});

API

dd(context)

Starts a drag interaction. Call it from a pointerdown (or mousedown) handler and pass the event in — dd throws without it. Document-level listeners are attached immediately (pointer events when the browser supports them, mouse events otherwise) and detached when the interaction ends or is canceled.

OptionDescription
eventRequired. The initiating PointerEvent or MouseEvent.
thresholdDistance in px the pointer must travel before begin/move/update start firing. Deltas keep measuring from the original press point — begin fires with dx/dy ≈ threshold, so a dragged item realigns with the pointer grip.
mixinsArray of handler objects. For every event, mixin handlers run first (in array order), then the context's own handler.

Handlers

Any of these can be defined on the context and on mixins. Each receives the context object.

HandlerWhen
beginInteraction started (after the threshold was crossed, if any).
movePointer moved.
updateAnything that requires a re-render: begin, pointer moved, or the page/a container scrolled. Usually the only handler you need.
endInteraction finished — released or canceled. Check ctx.canceled.
translateConvert client coordinates to your local space: read ctx.client_x/client_y, write ctx.x/ctx.y. Runs before deltas are computed.
begin_nothreshold, end_nothresholdRun unconditionally at press and release, even when the threshold was never crossed.

Context

FieldDescription
x, yCurrent position, after translate.
x0, y0Position at press.
dx, dyDistance traveled: x - x0, y - y0.
client_x, client_y, client_x0, client_y0, client_dx, client_dySame, in untranslated client coordinates.
eventThe last pointer event seen.
canceledtrue when the interaction was canceled: ESC, the window lost focus, or the browser took over the pointer (pointercancel, e.g. touch scrolling).

Built-in mixins

MixinDescription
dd.grid(n)Snap x/y to an n-pixel grid.
dd.prevent_default()preventDefault() on begin — stops text selection and native drag.
dd.no_pointer_events()Disables pointer events on the page during the drag and freezes the cursor, so hover styles and iframes don't interfere.

Debugging

dd.debug = true; // log begin/move/end transitions to the console

Notes

  • Touch devices: initiate from pointerdown and set touch-action: none on the drag handle — otherwise the browser takes over the gesture for scrolling and cancels the pointer.
  • Multi-touch: the interaction is bound to the initiating pointer; other fingers are ignored.
  • Old browsers: without PointerEvent support (IE, pre-2019 Safari) dd falls back to mouse events automatically.
  • Gotcha: IFRAME and BUTTON[disabled] swallow pointer events — dd.no_pointer_events() is the usual cure. (background)
